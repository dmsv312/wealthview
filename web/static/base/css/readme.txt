Добавление модальных окон

Сейчас на сайте 3 типа модальных окон, модальные окна справа (авторизации/регистрации и тд) модальное окно по центру большого размера (выбор актива) и модальное окно по центру меньшего размера (изменить портфель). Все 3 окна вызываются в common.js  

Модальное окно справа $js-modal,  модальные окна по центру любого размера вызываются $js-modal-inner. Размер модального окна по центру устанавливается с помощью добавления соответствующего класса b-modal-ctn (по дефолту оно большое) b-modal-ctn_sm маленькое. То есть при необходимости добавляем класс например b-modal-ctn_xs к блоку b-modal-ctn и уже у b-modal-ctn_xs прописываем нужную ширину или что то иное. Таким образом окна по центру можно модифицировать под разные нужды.

Для модальных окон справа существуют обязательные классы b-modal-default и mfp-hide. Класс mfp-hide обязателен для ВСЕХ типов модальных окон.

Для модальных окон по середине существует обязательный класс b-modal-ctn (он задаёт ширину модального окна) b-modal-ctn_sm (маленький размер окна)

Так же в common.js есть мобильное меню вызывается оно $js-modal-menu 

Что бы добавить модальное окно, необходимо:
1. если окно должно всплывать по ссылке, то для ссылки добавляем нужный класс соответствующий типу модального окна (js-modal/js-modal-inner)
2. в href ссылки мы добавляем id модального окна (пример #modal-auth) соответственно у нас на странице в шаблоне должен быть подключён/присутствовать блок c id modal-auth
3. Блоку с id modal-auth нужно обязательно добавить класс mfp-hide (этот класс скрывает по дефолту окно на странице) так же при необходимости добавить класс для стилизации данного модального окна (если оно отличается от уже существующих на сайте)
4. Если окно всплывает по нажатию на картинку или любой другой блок отличный от ссылки то добавляем этому объекту класс соответствующий типу модального окна (js-modal/js-modal-inner) и data-mfp-src=‘’ с соответствующим id


Добавление тултипов

Тултипы вызываются в common.js $js-tooltip

Что бы добавить тултип, необходимо:
1. В шаблон на страницу добавить блок с классом tooltip_templates и стилями style="display: none;" Это обертка в котором будут лежать тултипы или один тултип - не важно.
2. Положить тултип/блок с id в блок который мы создали ранее tooltip_templates
3. Что бы вызвать тултип нужно для любого объекта добавить class="js-tooltip" и data-tooltip-content="#" с соответствующим id 


Как работать со стилями

Файл который подключается у нас в шаблоне, это минифицированная версия (main.min.css) в который собраны все стили которые существуют на сайте (библиотеки плагинов и свои стили). 

Основной файл в котором собираются все стили это  main.css сюда импортируем все(библиотеки плагинов, свои стили)
Существуют некоторые категории на которые я разбил стили (base, layout и пр.) Base - здесь шрифты, кнопки и общие настройки проекта (типографика и прочее). Layout - здесь лежат стили касающиеся разметки сайта, каких то сеток, колонок, обёрток и пр.  Libs - здесь лежат стили подключенных библиотек. Modules - это части/блоки из которых состоит сайт (шапка, подвал и пр.) Webfonts - тут лежит икончатый шрифт fontawesome. 
Если стили которые вы собираетесь писать новые, то есть блок который вы описываете не существует на сайте, то создаете новый файл css с одноименным названием и размещаете в соответствующей папке категории (не забываем его подключить в main.css) и работаем в этом файле, в нем же пишем и медиа запросы (адаптив). Пишем стили касающиеся только одного этого блока, что то другое пишем уже в другом файле.
Если вы правите/добавляете стили для блоков которые уже есть на сайте, то ищем файл со стилями среди существующих и работаем в нем.

После того как вы поработали со стилями(изменили или добавили что то), нужно сделать компиляцию (собрать все стили в один файл, он должен называться main.min.css  и минифицировать его) удобным для вас способом.
